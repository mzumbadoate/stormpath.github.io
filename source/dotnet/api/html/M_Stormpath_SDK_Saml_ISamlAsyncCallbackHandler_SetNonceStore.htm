ISamlAsyncCallbackHandlerSetNonceStore Method
Sets a INonceStore that will retain message identifiers as cryptographic nonces, eliminating the possibility of a replay attack. This ensures any SAML Identity Provider response message cannot be intercepted and used again later.

Remarks

Because this ensures strong security, a INonceStore is enabled by default if you have caching enabled for the SDK: a cache region will be used to store nonces over time, and those nonces will be used to assert that SAML Identity Provider replies are not used more than once.

Default Nonce Store: A Cache-based INonceStore is enabled by default if you have caching enabled in the SDK. Because nonces are stored in a cache region, it is very important to ensure that the nonce store cache region has an entry TTL longer than the response message valid life span. For Stormpath, response messages are valid for 1 minute, so your default cache region settings must use a TTL longer than 1 minute (most caching system defaults are ~ 30 minutes or an hour, so odds are high that you're okay).

Custom Nonce Store: If you have not enabled caching in the SDK, or you don't want to use your SDK cache as a nonce store, you can specify a custom instance via this method, and all ID Site reply identifiers will be stored there to prevent reuse, but note: your custom INonceStore implementation MUST support the notion of a TTL (Time-to-Live) and automatically evict entries older than the max age lifespan (again, 1 minute).

If your INonceStore implementation does not support TTL auto-eviction, your store will fill up indefinitely, likely causing storage errors.
