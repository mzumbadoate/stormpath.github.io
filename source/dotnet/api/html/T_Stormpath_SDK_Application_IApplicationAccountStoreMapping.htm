Remarks
When an IApplicationAccountStoreMapping is created, the Accounts in the Account Store are granted access to become users of the linked Application. The order in which Account Stores are assigned to an Application determines how login attempts work in Stormpath.

Additionally, an IApplicationAccountStoreMapping may be designated as the default Account Store. This causes any accounts created directly by the Application to be dispatched to and saved in the associated Account Store, since Application cannot store Accounts directly.

Similarly, an IApplicationAccountStoreMapping may be designated as the default Group Store. This causes any Groups created directly by the Application to be dispatched to and saved in the associated Account Store, since an Application cannot store Groups directly. Note: A Group cannot store other Groups. Therefore, the default group store must be a Directory.
