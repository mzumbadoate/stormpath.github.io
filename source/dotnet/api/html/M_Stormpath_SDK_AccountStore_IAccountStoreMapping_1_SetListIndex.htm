Updates the zero-based order in which the associated Account Store will be consulted by the linked Application during an account authentication attempt.

Parameters

listIndex
Type: SystemInt32
If 0 is passed, the account store mapping will be the first item in the list. If 1 or greater is passed, the mapping will be inserted at that index. Because list indices are zero-based, the account store will be in the list at position listIndex - 1. If a negative number is passed, an ArgumentException will be thrown.

Remarks
If you use this setter, you will invalidate the cache for all of the associated Application's other AccountStoreMappings.
